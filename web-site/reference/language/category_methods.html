Methods

Prompto supports both global methods and category methods.

In this section, we describe global methods only, category methods are described in the Categories section.

A method is identified by its prototype, defined by its name and its accepted arguments.

Defining methods

A method is defined by declaring its prototype and optional return value, followed by the method statements.

The minimal prototype has the following form:

  • In E dialect: define methodName as method doing:
  • In O dialect: methodName() {}
  • In S dialect: def methodName():

Method names

A method name can be any valid identifier i.e. a word starting with a letter and followed by any number of letters or digits.

Method return types

A Prompto method can return any number of values.

If no value is returned, the return type is known as Void type.

If more than one value is returned, the return type is Tuple.

If exactly 1 value is returned, the return type can be specified. When not specified, it is inferred by Prompto.

Specifying the return type is done as follows:

  • In E dialect: define methodName as method returning Type doing:
  • In O dialect: Type methodName() {}
  • In S dialect: def methodName() -> Type:
where Type is the name of an existing type.

Method arguments

A Prompto method can accept 0 or more arguments.

Arguments must be declared (there is no implicit argument such as arguments in JavaScript).

Arguments are typed, in one of the following ways:

  • Attribute argument, with the form

    name, where name is an existing Attribute name.


  • Explicitly typed argument, with the form

    Type x, where type is an existing Type name and x can be any valid variable identifier.


  • Extended type argument, with the form

    Type x with attribute name, where type is an existing category name, x can be any valid variable identifier and name is an existing Attribute name. This form accepts more than one attribute.


  • Anonymous type argument, with the form

    any x with attribute name, where x can be any valid variable identifier and name is an existing Attribute name. This form accepts more than one attribute.


When mutiple arguments are declared, argument types can be mixed as required.

When mutiple arguments are declared, they must be separated by commas. In E dialect only, the last comma is replaced by the and keyword.

Method overloading

Prompto supports method overloading, where 2 methods have the same name but unrelated parameters.

When a method is overloaded, Prompto selects the most appropriate one based on the best match of the proposed parameters with respect to the required arguments. This matching if performed using the sequence of the parameteres in the method call.

Method overriding

Prompto supports method overriding, where a method can be overloaded using different but compatible parameters. Each overridden method provides a behavior specific to its argument types.

When a method is overridden, Prompto selects the most appropriate one at runtime i.e. the one for which the actual parameters best match the required method arguments, based on the sequence of the method call. This is different from many languages such as Java, C# or C++, where the resolution is decided at compile time.

Abstract methods

Prompto supports abstract methods, a special type of method, which must be overridden.

Abstract methods are resolved at runtime based on the actual type of the value.

Abstract methods are well suited when no actual behaviour can be determined for an argument type (such as Shape).

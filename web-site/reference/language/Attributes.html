Attributes

Prompto encourages technologists to use semantic types rather than technical ones.
A semantic type is a type defined by meaning rather than structure.
For example dateOfBirth is a semantic type, while Date is a technical type.
In Prompto, semantic types are defined through attribute definitions.
An attribute is defined by simply combining a name with a technical type.

Constraints

Additionally, attributes may also have to fit within value domains.
These can be defined using lexical patterns, lists of values or computable expressions.
Computable expressions can refer to the underlying value using the keyword value.

Storage

Finally, attributes are used to define the data store schema.
To achieve this, storable attribute definitions must be preceded with the storable keyword.
If the attribute is to be searchable, it must also be indexed.
For all technical types except Text, this is achieved by simply adding with index to the attribute definition.

Indexing Text

Indexing Text attributes requires a bit more attention.
Text attributes may be used as unique identifiers. Examples of such are a product reference or social security number.
Such Text attributes should be indexed using key indexing, which is case sensitive.
Most Text attributes are better indexed insensitively of case, and would happily benefir from a phonetic search.
Such Text attributes should be indexed using value indexing, which is not case sensitive.
(depending on the data store implementation, phonetic search may or not be available).
Finally, some Text attributes are actually sentences rather than 'atomic' values.
For these, it is often preferable to use words indexing.
Prompto lets you combine the 3 index types. It is rare however to simultaneously need more than 1 of them.

Built-in attributes

Character attributes

Character values have the following attributes:

  • codePoint: returns an Integer equal to the character 16-bit unicode code point

Text attributes

Text values have the following attributes:

  • count: returns an Integer equal to the number of characters

Date attributes

Date values have the following attributes:

  • year: returns an Integer equal to the date's year
  • month: returns an Integer equal to the date's month (January is 1, December is 12).
  • dayOfMonth: returns an Integer equal to the date's month day (1 to 31)
  • dayOfYear: returns an Integer equal to the date's year

Time attributes

Time values have the following attributes:

  • hour: returns an Integer equal to the time's hour (0 to 23)
  • minute: returns an Integer equal to the time's minute (0 to 59).
  • second: returns an Integer equal to the time's second (0 to 59)
  • millisecond: returns an Integer equal to the time's millisecond (0 to 999)

DateTime attributes

DateTime values have the following attributes:

  • year: returns an Integer equal to the date time's year
  • month: returns an Integer equal to the date time's month (January is 1, December is 12).
  • dayOfMonth: returns an Integer equal to the date time's month day (1 to 31)
  • dayOfYear: returns an Integer equal to the date time's year
  • hour: returns an Integer equal to the date time's hour (0 to 23)
  • minute: returns an Integer equal to the date time's minute (0 to 59).
  • second: returns an Integer equal to the date time's second (0 to 59)
  • millisecond: returns an Integer equal to the date time's millisecond (0 to 999)
  • tzOffset: returns an Integer equal to the date time's time zone offset in minutes
  • tzName: returns an Integer equal to the date time's time zone offset in minutes

Please note that due to limitations of Python and JavaScript, tzOffset and tzName are not currently supported on these platforms.

List attributes

List values have the following attributes:

  • count: returns an Integer equal to the number of items in the list

Set attributes

Set values have the following attributes:

  • count: returns an Integer equal to the number of items in the set

Tuple attributes

Tuple values have the following attributes:

  • count: returns an Integer equal to the number of items in the tuple

Dictionary attributes

Dictionary values have the following attributes:

  • count: returns an Integer equal to the number of key/value pairs in the dictionary
  • keys: returns a Set of Text values containing all the keys in the dictionary
  • values: returns a List of values containing all the values in the dictionary. The exact type of the List is determined by the exact type of the dictionary

Enumeration attributes

Enumeration values have the following attributes:

  • symbols: returns a List of the symbols in the enumeration
  • name: returns the name of the enumerated symbol as a Text value
  • value: returns the value represented by the enumerated symbol. The exact type of the value is determined by the exact type of the enumeration

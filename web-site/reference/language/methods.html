Methods

Prompto supports both global methods and category methods.

A method is identified by its prototype, defined by its name and the accepted arguments.

Prompto supports method overloading, where 2 methods have the same name but different parameters.

Prompto also supports category method overriding, where a method M define on category A is redefined on a category B which inherits from A, and provides a behavior specific to objects of category B.

Defining methods

A method is defined by declaring its prototype and optional return value, followed by the method statements.

  • In E dialect, the minimal prototype has the form define methodName as method doing:
  • In O dialect, the minimal prototype has the form methodName() {}
  • In P dialect, the minimal prototype has the form def methodName():

Method names

A method name can be any valid identifier i.e. a word starting with a letter and followed by any number of letters or digits.

Method return types

A Prompto method can return 0 or 1 value.

The return type can be specified. When not specified, it is inferred by Prompto.

  • In E dialect, the return type is specified using the returning keyword before the doing keyword as follows: define methodName as method returning Type doing:
  • In O dialect, the return type is specified before the method name Type methodName() {}
  • In P dialect, the return type is specified after the parenthesis and a right arrow def methodName() -> Type:
where Type is the name of an existing type.

Method arguments

A Prompto method can accept 0 or more arguments.

Arguments must be declared (there is no implicit argument such as arguments in JavaScript).

Arguments are typed, in one of the following ways:

  • Attribute argument, with the form

    name, where name is an existing Attribute name.


  • Explicitly typed argument, with the form

    Type x, where type is an existing Type name and x can be any valid variable identifier.


  • Extended type argument, with the form

    Type x with attribute name, where type is an existing category name, x can be any valid variable identifier and name is an existing Attribute name. This form accepts more than one attribute.


  • Anonymous type argument, with the form

    any x with attribute name, where x can be any valid variable identifier and name is an existing Attribute name. This form accepts more than one attribute.


When mutiple arguments are declared, argument types can be mixed as required.

When mutiple arguments are declared, they must be separated by commas. In E dialect only, the last comma is replaced by the and keyword.

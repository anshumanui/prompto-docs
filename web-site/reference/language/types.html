Prompto built-in types

Like most languages, Prompto comes with built-in types for commonly used types of values.
Atomic types are built-in types which can hold a single 'atomic' value: text, number, date and so forth. Collection types are built-in types which can hold multiple values, typically of the same atomic type.
Special types are built-in types which have no textual representation, but are useful for manipulating key programming objects
Prompto also supports composed types, however these are user defined (although Prompto libraries may define some). See attributes and categories for the definition of composed types.

Atomic types

Atomic types are built-in types which can only hold a single 'atomic' value.

Text

The Text built-in type is used for values such as a name or description, where the textual representation and the meaning are equivalent.
All Text values use Unicode internally.
A Text literal must be enclosed in double quotes.

"This is a text literal."

If a Text literal must contain a double-quote, that double-quote must be escaped using the anti-slash '\' character.

"This text literal contains two \"double quotes\"."

Consequently, if a Text literal must contain the anti-slash'\' character, the '\' character must be escaped using the '\' character.

"This text literal contains one anti-slash: \\."

Additionally, Text literals support the following special characters:

"\t" -> Tab
"\b" -> Back space
"\n" -> New line
"\r" -> Carriage return
"\f" -> Form feed

Finally, Text literals support Unicode characters in the form of UTF-16 hexadecimal characters:

"delta: \u0394" -> delta: δ

Character

The Character built-in type is used for easy manipulation of single characters.
Like Text values, all Character values use Unicode internally.
Character literals support the same escape characters as Text literals.
A Character literal must be enclosed in single quotes.

'c' -> the 'c' character

Boolean

The Boolean built-in type is used exclusively for the values true and false.
A Boolean literal is simply one of the words true or false.
Alternately, Prompto also supports the words True and False.

Integer

The Integer built-in type is used for integral numbers, using 8 bytes, thus ranging from -263 to 263-1.
An Integer literal is simply a sequence of digits, such as 13465.
Prompto also supports Integer literals using hexadecimal notation, such as 0xA3B2FE.
Finally, Prompto supports 2 special Integers, MIN_INTEGER and MAX_INTEGER.

Decimal

The Decimal built-in type is used for non integral numbers, using 8 bytes as specified by IEE 754.
A Decimal literal is simply a sequence of digits, followed by a dot and another sequence of digits, such as 13465.02.
Prompto also supports Decimal literals using scientific notation, such as 0.074e32.

Time

The Time built-in type is used for to represent a time of day. It does not support time zones.
A Time literal is a sequence of 2 to 4 integers, separated by a semi colon ':' or a dot for the last one, representing the hour of day, the minute, the optional second and the optional millisecond.
A Time literal conforms to the ISO 8601 format.
A Time literal must be enclosed in single quotes.

'13:07' -> 13 hours and 7 minutes
'13:07:02.013' -> 13 hours, 7 minutes, 2 seconds and 13 milliseconds

Date

The Date built-in type is used to represent a day. It does not support time zones.
A Date literal is a sequence of 3 integers, separated by a hyphen '-', representing the year, the month and the day of the month.
A Date literal conforms to the ISO 8601 format.
A Date literal must be enclosed in single quotes.

'1981-05-12' -> 12th of May of year 1981

DateTime

The DateTime built-in type is used for to represent an instant, with time zone support.
A DateTime literal is formed by a Date literal, followed by a 'T', followed by a Time literal, optionally followed by a time zone offset.
A DateTime literal conforms to the ISO 8601 format.
A DateTime literal must be enclosed in single quotes.

'1981-05-12T13:02:11.002' -> 13 hours, 2 minutes, 11 seconds and 2 milliseconds on the 12th of May of year 1981
'1981-05-12T13:02:11.002+0200' -> same date time as above, in the GMT + 02:00 time zone

Period

The Period built-in type is used for to represent a period of time, or a duration.
A Period literal is formed by a 'P' followed by a sequence of integral quantities preceded by their unit, see examples.
Day level units and time level units are separated by a 'T', as in DateTime literals.
A Period literal conforms to the ISO 8601 format.
A Period literal must be enclosed in single quotes.

'PY3W2' -> 3 years + 2 weeks
'PM4D1' -> 4 months + 1 day
'PD1TH2' -> 1 day + 2 hours
'PTM4S173' -> 4 minutes + 173 seconds

Blob

The Blob (Binary Large OBject) built-in type is a type used to hold an arbitray list of raw bytes.
A Blob has a mime type, which is useful to describe the type of the binary data, notably over the Internet.
There is no Blob literal.

Image

The Image built-in type is a specialized Blob type used to hold displayable bynary data, such a JPEG image.
An Image has a mime type, which is useful to describe the type of the binary data, notably over the Internet.
There is no Image literal.

UUID

The UUID built-in type is a type often used in IT systems such as databases, to provide unique identifiers.
A UUID generator is a system which using specific hardware unique data such as MAC address, and current instant, is able to generate a globally Universal Unique IDentifier.
A UUID literal is a sequence of characters, digits and hyphens which conform to the RFC 4122 specification.
A UUID literal must be enclosed in single quotes.

'123e4567-e89b-12d3-a456-426655440000' -> a valid UUID

Collection types

Collection types are built-in types which can hold multiple values, typically of the same atomic type.

List

The List built-in type is used to keep together 0 or more items of the same type.
The items themselves can be of any type.
A List literal must be enclosed in square brackets '[' and ']', the items being separated by a comma ','.

[12, 15, -3] -> a List of 3 Integers
["Paris", "London", "Singapore"] -> a List of 3 Texts

Items in a List can be accessed individually using their position.
Positions start at 1 and end at the list length.

a = [12, 15, -3] -> assign a List of 3 Integers to variable 'a'
a[1] -> get item at position 1 in List referenced by variable 'a' i.e. the Integer 12

Tuple

The Tuple built-in type is used to keep together 0 or more items of different types.
The items themselves can be of any type.
A Tuple literal must be enclosed in parenthesis '(' and ')', the items being separated by a comma ','.
For Tuple comprising 0 or 1 item, because the above could be interpreted as a parenthesized expression, a comma ',' must precede the right parenthesis.

(,) -> the empty Tuple (3.14,) -> a Tuple of 1 item
("Paris", '2015-06-12', -113.87) -> a Tuple of 3 items

Items in a Tuple can be accessed individually using their position.
Positions start at 1 and end at the list length.

a = ("Paris", '2015-06-12', -113.87) -> assign a Tuple of 3 items to variable 'a'
a[1] -> get item at position 1 in Tuple referenced by variable 'a' i.e. the Text "Paris"

Set

The Set built-in type is used to keep together 0 or more distinct items of the same type.
The items themselves can be of any type.
A Set literal must be enclosed in angle brackets '<' and '>', the items being separated by a comma ','.

<12, 15, -3> -> a set of 3 Integers
<"Paris", "London", "Singapore"> -> a Set of 3 Texts

Items in a Set can be accessed individually using their position.
However this is discouraged, because the position in the Set may change internally when ensuring uniqueness.
Set items are generally accessed using iterators or predicates.

s = <12, 15, -3> -> assign a Set of 3 Integers to variable 's'
for each i in s -> iterate over items in Set referenced by variable 's'
s contains 12 -> return true if Set referenced by variable 's' contains the Integer 12

Dictionary

The Dictionary built-in type is used to keep together 0 or more items of the same type and access them using a Text key.
The items themselves can be of any type.
A Dictionary literal must be enclosed in curly brackets '{' and '}', the items being separated by a comma ',', the key and value being separated by a semi colon ':'.

{} -> an empty Dictionary
{ "Paris" : "France", "London" : "England", "Berlin" : "Germany" } -> a Dictionary of 3 keys and Text values

Items in a Dictionary can be accessed individually using their key.

d = { "Paris" : "France", "London" : "England" } -> assign a Dictionary of 2 Texts to variable 'd'
d["Paris"] -> get item with key "Paris" i.e. Dictionary referenced by variable 'd' i.e. "France"

Document

The Document built-in type is used to manipulate arbitrary data structures with loose typing.
A Document has items which can be of any type, atomic, collection or Document.
A Document is useful to carry data read from the store, from files or from the Internet, and convert it to typed data.
A Document is also useful to write typed data to the store, to files or to the Internet.

Range

The Range built-in type is used to create sequences of 1 or more items of the same discrete type.
The items themselves can only be of type Character, Integer, Date or Time.
A Range literal must be enclosed in square brackets '[' and ']', the start and end value being separated by 2 dots '..'.

[1..17] -> all Integers ranging from 1 to 17 included
['c'..'k'] -> all Characters ranging from 'c' to 'k' included (using their Unicode value)
['2012-01-01'..'2014-04-01'] -> all Dates ranging from '2012-01-01' to '2014-04-01'
['20:01:01'..'23:02:02'] -> all Times ranging from '20:01:01' to '23:02:02', with a 1 second increment

Items in a Range can be accessed individually using their position.
Positions start at 1 and end at the Range length.

r = [2..17] -> assign a Range of Integers to variable 'r'
r[4] -> get item at position 4 in Range referenced by variable 'r' i.e. the Integer 5

Special types

Special types are built-in types which are used in specific situations.

Any

The Any built-in type is a general type to represent any type.
This type is useful for describing parameter types by attribute.
It can also be useful to hold typed collections where the type is not known at compile time.
It is also useful to represent opaque native types.

Method

The Method built-in type is a type used to hold Prompto methods.
Prompto supports passing methods as parameters.

m = myMethod -> assign the method 'myMethod' to variable 'm'

Code

The Code built-in type is a type used to represent Prompto code.
Prompto supports passing code snippets as parameters, so this is useful for passing code to methods.

c = Code: a + b -> assign the Prompto code 'a + b' to variable 'c'

Iterator

The Iterator built-in type is a type produced by Prompto when transforming a collection inline.

x + 1 for each x in [1..5] -> returns an Iterator which will produce an item on each iteration

Cursor

The Cursor built-in type is a type used to navigate through data returned by the data store.

c = fetch all -> returns a Cursor which will produce an item on each iteration

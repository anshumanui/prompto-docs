Types

This section describes types in the Prompto language.

Programming languages are used to manipulate all sorts of values: text, image, numbers, time...
Most languages associate values with types. This is required to determine which operations can be performed on a value, and how.
Values can be provided using inline literals, like 123 or "John".
Values can also be read from a database, files, the internet and so forth.
In both cases, manipulating values is often done using variables, which associate a name with a value.
Writing a = 123 binds the name a with the integer 123.
As a consequence, the type of a is integer.
Prompto is statically typed, which means that the type of a variable cannot change.
Prompto uses type inference, which means that generally, types only need to be declared for method parameters.

Prompto technical types

Prompto comes with built-in types for commonly used types of values.
Prompto calls them technical types, because they are defined by their technical structure rather than their meaning.
Value types are technical types which can hold a single 'atomic' value: text, number, date and so forth. Collection types are technical types which can hold multiple values, typically of the same atomic type.
Special types are technical types which do not contain information but are useful during program execution

Prompto also supports semantic types and composed types, however these are user defined (although Prompto libraries may define some).
See attributes and categories for the definition of semantic and composed types.

Being statically typed, Prompto sometimes requires that the type of an object be specified.
This is achieved by simply providing the type name.
The exact syntax depends on the dialect and the fragment being defined.
The below is an example of an attribute declaration in the E dialect. 

define name as Text attribute

Value types

Value types are technical types which can only hold a single 'atomic' value.

Text

The Text type is a technical type used for values such as a name or description, where the textual representation and the meaning are equivalent.
All Text values use Unicode internally.
A Text literal must be enclosed in double quotes.

"This is a text literal."

If a Text literal must contain a double-quote, that double-quote must be escaped using the anti-slash '\' character.

"This text literal contains two \"double quotes\"."

Consequently, if a Text literal must contain the anti-slash'\' character, the '\' character must be escaped using the '\' character.

"This text literal contains one anti-slash: \\."

Additionally, Text literals support the following special characters:

"\t" -> Tab
"\b" -> Back space
"\n" -> New line
"\r" -> Carriage return
"\f" -> Form feed
"\'" -> Single quote

Finally, Text literals support Unicode characters in the form of UTF-16 hexadecimal characters:

"delta: \u0394" -> delta: δ

Character

The Character type is a technical type used for easy manipulation of single characters.
Like Text values, all Character values use Unicode internally.
Character literals support the same escape characters as Text literals.
A Character literal must be enclosed in single quotes.

'c' -> the 'c' character

Boolean

The Boolean type is a technical type used exclusively for the values true and false.
A Boolean literal is simply one of the words true or false.
Alternately, Prompto also supports the words True and False.

Integer

The Integer type is a technical type used for integral numbers, using 8 bytes, thus ranging from -263 to 263-1.
An Integer literal is simply a sequence of digits, such as 13465.
Prompto also supports Integer literals using hexadecimal notation, such as 0xA3B2FE.
Finally, Prompto supports 2 special Integers, MIN_INTEGER and MAX_INTEGER.

Decimal

The Decimal type is a technical type used for non integral numbers, using 8 bytes as specified by IEE 754.
A Decimal literal is simply a sequence of digits, followed by a dot and another sequence of digits, such as 13465.02.
Prompto also supports Decimal literals using scientific notation, such as 0.074e32.

Please note that due to their internal binary representation, not all Decimal values can be accurately represented.
For example, the internal representation for 1.3 is actually 1.2999999999999998.
See this detailed explanation with more examples.

Time

The Time type is a technical type used for to represent a time of day. It does not support time zones.
A Time literal is a sequence of 2 to 4 integers, separated by a semi colon ':' or a dot for the last one, representing the hour of day, the minute, the optional second and the optional millisecond.
A Time literal conforms to the ISO 8601 format.
A Time literal must be enclosed in single quotes.

'13:07' -> 13 hours and 7 minutes
'13:07:02.013' -> 13 hours, 7 minutes, 2 seconds and 13 milliseconds

Date

The Date type is a technical type used to represent a day. It does not support time zones.
A Date literal is a sequence of 3 integers, separated by a hyphen '-', representing the year, the month and the day of the month.
A Date literal conforms to the ISO 8601 format.
A Date literal must be enclosed in single quotes.

'1981-05-12' -> 12th of May of year 1981

DateTime

The DateTime type is a technical type used to represent an instant, with time zone support.
A DateTime literal is formed by a Date literal, followed by a 'T', followed by a Time literal, optionally followed by a time zone offset.
A DateTime literal conforms to the ISO 8601 format.
A DateTime literal must be enclosed in single quotes.
If a time zone offset is not specified, the literal is interpreted in the time zone of the eoperating system.

'1981-05-12T13:02:11.002' -> 13 hours, 2 minutes, 11 seconds and 2 milliseconds on the 12th of May of year 1981
'1981-05-12T13:02:11.002+0200' -> same date time as above, in the GMT + 02:00 time zone

Period

The Period type is a technical type used to represent a period of time, or a duration.
A Period literal is formed by a 'P' followed by a sequence of integral quantities preceded by their unit, see examples.
Day level units and time level units are separated by a 'T', as in DateTime literals.
A Period literal conforms to the ISO 8601 format.
A Period literal must be enclosed in single quotes.

'PY3W2' -> 3 years + 2 weeks
'PM4D1' -> 4 months + 1 day
'PD1TH2' -> 1 day + 2 hours
'PTM4S173' -> 4 minutes + 173 seconds

Document

The Document type is a technical type used to manipulate arbitrary data structures with loose typing.
A Document has items which can be of any type, atomic, collection or Document.
A Document is useful to carry data read from the store, from files or from the Internet, and convert it to typed data.
A Document is also useful to write typed data to the store, to files or to the Internet.

Blob

The Blob (Binary Large OBject) type is a technical type used to hold an arbitray list of raw bytes.
A Blob has a mime type, which is useful to describe the type of the binary data, notably over the Internet.
There is no Blob literal.

Image

The Image type is a specialized Blob type used to hold displayable binary data, such as a JPEG image.
An Image has a mime type, which is useful to describe the type of the binary data, notably over the Internet.
There is no Image literal.

UUID

The UUID type is a technical type often used in IT systems such as databases, to provide unique identifiers.
A UUID generator is a system which using specific hardware unique data such as MAC address, and current instant, is able to generate a globally Universal Unique IDentifier.
A UUID literal is a sequence of characters, digits and hyphens which conform to the RFC 4122 specification.
A UUID literal must be enclosed in single quotes.

'123e4567-e89b-12d3-a456-426655440000' -> a valid UUID
